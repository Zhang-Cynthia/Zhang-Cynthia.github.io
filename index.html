DockerLearning

Posted on 2024-06-05 Edited on 2024-06-06

1 基本概念

Docker官方文档

Docker官网下载安装

1.1 镜像（Image）

Docker镜像就像一个特殊的文件系统，可以提供容器运行时所需的程序、库、资源、配置等等，通过这个镜像可以创建多个容器。

1.2 容器（Container）

容器可以被创建、启动、停止、删除、暂停等，Docker利用容器技术，独立运行一个或者一组应用。

1.3 仓库（Repository）

仓库就是存放镜像的地方，每个仓库可以包含多个标签（Tag），每个标签对应一个镜像。通常一个仓库会包含同一个软件的不同版本的镜像，而标签就常用于对应该软件的各个版本，如果不给定标签，将以latest作为标签。

2 常用命令

docker images   #查看所有本地的主机上的镜像
docker search 镜像名 # 搜索镜像

2.1 下载镜像

docker pull 镜像名:版本号    #不写版本号默认下载最新版latest
docker pull ubuntu:24.04  #例如拉取ubuntu镜像

2.2 删除镜像

docker rmi -f 容器id   #-f就是全部删除，后面的条件是根据什么来删除
#删除多个镜像
docker  rmi -f  容器id  容器id 容器id    

# 删除所有的镜像 
docker  rmi -f $(docker images -aq)

2.3 新建容器并启动

docker run [可选参数] image
参数说明：
--name="Name" ： 容器名字，tomcat01、tomcat02，用来区分容器
-d ：后台方式运行                                
-it：使用交互方式运行，进入容器查看内容
-p ：指定容器的端口   -p  8080:8080  
        -p     ip:主机端口:容器端口                                
        -p     主机端口：容器端口（常用方式）
        -p     容器端口                                                     
        容器端口                                                             
-p： 随机指定端口
#启动容器
docker run -it hexo /bin/bash
#从容器钟退回到主机，直接停止容器并退出
exit
#进入当前正在运行的容器
docker exec -it 容器id bashShell
#例如进入hexo容器
docker exec -it hexo /bin/bash

本地连接到Github

Posted on 2024-06-05

1 启动容器

确保docker里的hexo已经打开，exec执行

docker exec -it hexo /bin/bash

2 设置用户名和邮箱

注意复制用户名和邮箱的时候引号里面不要有空格

git config --global user.name "GitHub 用户名"
git config --global user.email "GitHub 邮箱"

3 创建SSH密钥

ssh-keygen -t rsa -C "GitHub 邮箱"

然后一路回车即可

3.1 添加密钥

进入root/.ssh目录，cat打开id_rsa.pub文件并复制里面的内容，如果出现乱码，则可能是第二步设置用户名和邮箱的时候粘贴了空格，重新执行一遍即可

cd root/.ssh
cat id_rsa.pub

登录Github，进入settings页面，选择左边栏的SSH and GPG keys，点击 New SSH key。Title 随便取个名字，粘贴复制的 id_rsa.pub 内容到 Key 中，点击 Add SSH key 完成添加。

3.2 验证连接

ssh -T git@github.com

输入yes回车确认，显示 “Hi xxx! You’ve successfully……” 即连接成功。

4 创建Github Pages仓库

GitHub 主页右上角加号 -> New repository：

  • Repository name 中输入 用户名.github.io #用户名是Github用户名
  • 勾选 “Initialize this repository with a README”
  • Description 选填

填好后点击创建，创建后默认自动启用 HTTPS，博客地址为：https://用户名.github.io

5 部署Hexo到Github Pages

首先安装hexo-deployer-git

npm install hexo-deployer-git --save

然后进入hexo/blog里的_config.yml文件，将末尾的内容修改成如下内容。注意type、repository和branch冒号后面有一个空格。

deploy:
  type: git
  repository: git@github.com:用户名/用户名.github.io.git
  branch: gh-pages

其实就是通过 SSH 进行连接，SSH 连接地址可以 直接在仓库 <>Code 里面选择看到。

Ctrl+S保存后，

hexo clean  #清除缓存和已生成的静态文件
hexo new "My First Blog" #新建md文件
hexo g   #生成页面
hexo s    #启动预览
hexo d    #将网站上传到Github上

然后需要在库设置（Repository Settings）中将默认分支设置为_config.yml配置中的分支名称，即gh-pages

然后在 pages 页面调整源分支：

这时浏览器访问 https://用户名.github.io 就可以看到 Hexo 网站了。
